# OpenMaple Support

OpenMaple is an open-source project. Support is community-driven and should leave useful evidence for the next team evaluating the platform.

## Where To Ask

- **Questions and design discussion**: use [GitHub Discussions](https://github.com/dragonforce2010/openmaple/discussions).
- **Reproducible bugs**: open a [bug report](https://github.com/dragonforce2010/openmaple/issues/new?template=bug_report.yml).
- **Feature requests**: open a [feature request](https://github.com/dragonforce2010/openmaple/issues/new?template=feature_request.yml).
- **Security vulnerabilities**: follow [SECURITY.md](SECURITY.md). Do not file public issues for secrets, auth bypasses, or remote-code-execution reports.

## Good Support Requests

Include:

- OpenMaple commit or release version.
- Runtime path: local dev, Docker Compose, veFaaS, E2B, Docker sandbox, or another provider.
- Commands run and full error output.
- Relevant `.env` variable names with secret values removed.
- Screenshots or session event IDs when debugging console or session behavior.

## Scope

The project can help with OpenMaple code, examples, provider contracts, SDK/CLI usage, and documentation gaps. It cannot provide managed production operations, cloud account administration, or support for private forks that cannot share enough detail to reproduce the issue.
